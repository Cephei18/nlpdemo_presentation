1. Problem Statement

Traditional automated graders create a double penalty for dyslexic students:

  • Spelling Penalty Misspelled keywords flagged as incorrect despite correct concepts.
  • Semantic Failure Standard BERT models fail to tokenize "noisy" text, losing context.

2. Dataset & Data Engineering

Strategic Data Scaling

Phase 1 (M0 Experiment): Initially, we collected 100 real-world handwritten samples to train a custom OCR model. Due to data scarcity, accuracy was low (<60%).

Strategic Pivot: Instead of discarding this data, we analyzed the error patterns (transpositions, phonetic swaps) within it to engineer a robust Data Generator. This allowed us to synthesize a massive, high-quality training corpus.

3. Methodology (M1 ‚Üí M2)

M0: Acquisition

Robust entry handling. Upgraded from custom OCR to Google Cloud Vision API integration / Simulation for stability.

M1: Normalizer

Damerau-Levenshtein Distance. Specifically targets transpositions (e.g., 'teh' ‚Üí 'the'), a key dyslexic marker.

M2: Semantic Grader

Fine-Tuned DistilBERT. Trained for 40 Epochs to learn Semantic Textual Similarity (STS).
